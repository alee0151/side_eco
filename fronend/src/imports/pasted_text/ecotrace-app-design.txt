Design a complete responsive web application for EcoTrace, an Australian biodiversity intelligence platform that helps everyday Australians and investors understand a company’s biodiversity impact. The product should feel trustworthy, clean, eco-friendly, and modern, with a premium SaaS aesthetic. Use a soft cream or white background, green and blue palette, rounded cards, subtle shadows, generous whitespace, and flat vector icons. The UI must be simple enough for everyday users but detailed enough for investors and ESG professionals.
The app should support three main user types:
Everyday consumers searching by barcode, brand, or company name.
Investors using reports, watchlists, and alerts.
ESG/b2b users using an API and dashboard.
Create the design as a multi-page Figma file with the following structure:
Design system page with colors, typography, spacing, buttons, cards, chips, badges, tabs, tables, forms, and alert states.
Page 1: Consumer search and entity resolution flow.
Page 2: Company overview and biodiversity risk results.
Page 3: Analyse page with evidence, locations, and map-based risk.
Page 4: Knowledge graph and source provenance view.
Page 5: Supply chain intelligence and supplier mapping.
Page 6: Investor dashboard with TNFD report, watchlist, and alerts.
Page 7: Admin / pipeline monitoring for data ingestion and QA.
Page 8: B2B API and export page.
Page 9: Settings, permissions, and report history.
The UI should make biodiversity risk easy to understand in under 30 seconds for consumers, while giving investors a detailed, auditable, TNFD-aligned view. Use a left sidebar for navigation in desktop, a top search bar, and a right-side evidence drawer in detailed views. Show realistic dashboard content, charts, maps, evidence cards, confidence indicators, audit trail markers, and biodiversity badges. Include empty states, loading states, error states, and confirmation modals.
For the consumer experience, show barcode scanning, brand search, company search, a biodiversity risk score from 0–100, green/yellow/red badges, and a “Better Choice” alternative recommendation panel. For the investor experience, show TNFD LEAP sections, watchlists, portfolio alerts, evidence timelines, and export to PDF. For the supply chain experience, show subsidiary relationships, supplier nodes, and a risk propagation view. For the provenance experience, show source documents, confidence scores, timestamps, and traceable claim chains.
Use a clean, highly legible typography system, accessible contrast, and polished components. The overall product should look like a real venture-backed SaaS platform, not a concept mockup. Build it as a professional, production-ready design system and app structure.
Page-by-Page Figma Structure
0. Design System
This page should be the foundation for the whole product.
Include:
Color palette: primary green, secondary blue, accent yellow/orange, neutrals, success/warning/error states.
Typography scale: H1, H2, H3, body, caption, labels.
Buttons: primary, secondary, ghost, danger.
Badges: low risk, medium risk, high risk, confidence, verified, draft.
Cards: simple, hover, evidence, KPI, alert cards.
Navigation styles: sidebar, top bar, tabs, breadcrumbs.
Form elements: search input, dropdown, multi-select, toggle, radio, checkbox.
Tables and filters.
Chart styles and icon rules.
Spacing system and grid rules.
Status states: loading, empty, error, success.
Purpose:
Ensures consistent visuals across all 6 epics and makes the system scalable.
